# Security

This is a static marketing site (HTML / CSS / JS, no server, no database, no
user data collected). The realistic attack surface is small — but if you find
something, we still want to hear about it.

## Reporting a vulnerability

Email **jnew00@gmail.com** with:

- A short description of the issue
- Steps to reproduce
- Any suggested mitigation

Please do not open a public GitHub issue for security reports.

We'll acknowledge within a few days and follow up with a fix or an
explanation of why it isn't actionable.

## Scope

In scope:

- The HTML / CSS / JS shipped from this repository
- The assets served from `assets/`

Out of scope:

- Vulnerabilities in third-party services (Google Fonts, etc.)
- The FoodieMerge mobile game itself (separate repo, separate security policy)
- DoS or volumetric attacks against the hosting provider
